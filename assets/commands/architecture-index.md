Generate or refresh the architecture knowledge-base hub at `docs/architecture/INDEX.md`.

Arguments: $ARGUMENTS

This is a navigational pass. It produces the single entry point a new engineer opens first. It is idempotent — re-run any time to refresh after other docs change.

## Tasks

1. Read every file in `docs/architecture/`.
2. For each artifact, determine its status:
   - **complete** — the doc has its expected sections filled, confidence labels present, Mermaid diagrams with `click` directives, and a backlinks section.
   - **partial** — doc exists but is missing one or more expected sections.
   - **missing** — expected artifact not present on disk.
   - **stale** — doc references file paths that no longer exist (spot-check the top 5-10 citations per doc).
3. Write `docs/architecture/INDEX.md` using the structure below.

## Rules

- Pull all counts and summaries from the existing docs — do not re-analyze source.
- Status markers must be honest; under-report rather than over-report. A "complete" label is a claim the doc meets the linking contract.
- If `system-flowchart.md` is missing, say so prominently at the top of the index — it's the headline artifact.
- Leave the "Top findings" section empty (rather than filled with filler) if nothing clearly rises. Empty beats generic.
- Preserve any user-added content in INDEX.md between `<!-- USER-CONTENT-START -->` and `<!-- USER-CONTENT-END -->` markers if present. Regenerate everything else.
